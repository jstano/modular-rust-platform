# stano-seaorm

Database layer utilities for Postgres via SeaORM: connection pool configuration and a `Mapper` trait for bidirectional domain ↔ ORM conversion.

## Install

```toml
[dependencies]
stano-seaorm = { path = "../stano-seaorm" }
sea-orm = { version = "1.1", features = ["runtime-tokio-rustls", "sqlx-postgres"] }
```

## API

### Database Configuration

- **`DbConfig`** — wraps a SeaORM `DatabaseConnection`.
  - `async fn from_url(database_url: &str) -> Result<Self, DbConfigError>` — create a pool from a Postgres URL. Hardcoded settings:
    - **Max connections:** 100
    - **Min connections:** 5
    - **Connect timeout:** 30 seconds
    - **Acquire timeout:** 30 seconds
    - **Idle timeout:** 10 minutes
    - **Max lifetime:** 1 hour
  - `fn connection(&self) -> &DatabaseConnection` — retrieve the underlying SeaORM connection.

- **`DbConfigError`** — error type:
  - `ConnectionFailed(String)` — failed to connect to the database.

### Entity Mapping

- **`Mapper<Domain>`** — trait for bidirectional domain ↔ ORM conversion.
  ```rust
  pub trait Mapper<Domain> {
      type Model;                         // SeaORM entity Model type
      type ActiveModel;                   // SeaORM entity ActiveModel type
      fn to_domain(model: Self::Model) -> Domain;
      fn to_active_model(domain: &Domain) -> Self::ActiveModel;
  }
  ```

### Re-exports

- `sea_orm` crate — for entity definitions, queries, migrations, etc.

## Usage Example

```rust
use stano_seaorm::{DbConfig, Mapper};
use sea_orm::{ColumnTrait, EntityTrait, QueryFilter};

// 1. Set up the database connection.
let db = DbConfig::from_url("postgres://user:pass@localhost/mydb").await?;
let conn = db.connection();

// 2. Define your domain entity (in your app's domain crate).
pub struct User {
    pub id: uuid::Uuid,
    pub email: String,
}

// 3. Define a SeaORM entity (generated by sea-orm-cli or hand-written).
use sea_orm::prelude::*;

#[derive(Clone, Debug, DeriveEntityModel)]
#[sea_orm(table_name = "users")]
pub struct UserModel {
    #[sea_orm(primary_key)]
    pub id: Uuid,
    pub email: String,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {}

impl ActiveModelBehavior for UserModel {}

// 4. Implement the Mapper trait.
pub struct UserMapper;

impl Mapper<User> for UserMapper {
    type Model = UserModel;
    type ActiveModel = user::ActiveModel;

    fn to_domain(model: UserModel) -> User {
        User {
            id: model.id,
            email: model.email,
        }
    }

    fn to_active_model(domain: &User) -> user::ActiveModel {
        user::ActiveModel {
            id: Set(domain.id),
            email: Set(domain.email.clone()),
        }
    }
}

// 5. Use in a repository.
pub async fn find_user_by_id(id: uuid::Uuid, conn: &DbConnection) -> Result<Option<User>, DbErr> {
    user::Entity::find_by_id(id)
        .one(conn)
        .await
        .map(|opt| opt.map(UserMapper::to_domain))
}
```

## Notes

- **Pool settings are hardcoded** — no per-app customization. If different timeouts or pool sizes are needed, consider wrapping `DbConfig` in your app.
- **SeaORM re-export** — full `sea_orm` crate is re-exported; use it for entity generation, query construction, and migrations.
- **ORM migration** — `Mapper` is a convention, not middleware; you must manually implement it for each entity to bridge domain and database layers.
- **No feature flags** — Postgres via sqlx is always enabled (not optional).
